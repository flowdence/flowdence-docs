## Purpose

Define ownership, review frequency, and release gates for shared and app-specific public documentation.

## Ownership model

- Legal documents: Legal owner with product co-review.
- Security and support policies: Security/support owner with engineering co-review.
- Product documentation: Product owner with engineering validation.

## Required review events

1. Quarterly policy review.
2. Before any Marketplace submission or major listing update.
3. After any security incident or major support process change.
4. After material architectural changes affecting claims.

## Minimum metadata standard

Every public markdown document must include:

- Last updated date
- Owner
- Applies to app
- Review cadence

## Change control

- Material policy changes require decision log entry in each affected app repository.
- Shared baseline changes must be communicated to all app owners.
